# plotto Hardware Specification

## 1. 개요

본 문서는 로봇팔 시스템 **plotto**의 하드웨어 구성과  
소프트웨어 제어를 위해 정의된 **기구학적 기준 및 제원**을 설명합니다.

해당 문서는 관절 구조, 좌표계 정의, 각도 원점, 가동 범위 등  
소프트웨어에서 로봇 동작을 생성하고 해석하기 위해 필요한  
**하드웨어 기준 정보**를 제공하는 것을 목적으로 합니다.

---

## 2. 로봇 하드웨어 구성 개요

### 2.1. 로봇 타입

- 로봇 형태: **2-DOF Drawing Robot Arm**
- 구동 방식: 회전 관절 기반으로 동작

### 2.2. 로봇 구성

|  Joint 2  | 팔꿈치 관절 |

---

## 3. 좌표계 정의

<img width="575" height="365" alt="Image" src="https://github.com/user-attachments/assets/9ee912d2-ee7e-4afe-9892-14c4d9b43279" />

원점 (0, 0): 로봇의 기본 위치로, Joint 1과 Joint 2의 각도가 모두 0°인 상태 (이미지 참고)

모든 관절 각도는 **시계 반대 방향(CCW)** 을 양(+)의 방향으로 정의합니다.

각도 단위는 **degree(°)** 를 기준으로 합니다.

---

## 4. 관절 가동 범위 (Joint Limit)

각 관절은 기구학적·물리적 제약에 의해 가동 범위가 제한됩니다.

|  관절   | 최소 각도 | 최대 각도 |
| :-----: | :-------: | :-------: |
| Joint 1 |   -30°    |   180°    |
| Joint 2 |   -100°   |    15°    |

> 실제 하드웨어 확정 이후 값이 갱신될 수 있습니다.

> 소프트웨어는 해당 범위를 벗어나는 명령을 생성하지 않도록 제한 영역 설정이 필요합니다.

---

## 5. 펜 메커니즘

### 5.1. 펜 상태 정의

|   상태   | 설명                                    |
| :------: | :-------------------------------------- |
|  Pen Up  | 펜이 종이에서 떨어진 상태 (이동만 수행) |
| Pen Down | 펜이 종이에 접촉한 상태 (드로잉 수행)   |

### 5.2. 소프트웨어 제어 기준

펜 상태는 관절 동작과 독립적인 제어 명령으로 처리됩니다.

plotto Path 포맷에서는 별도의 Control Byte로 표현됩니다.

자세한 구조는 [plotto Path](plotto_Path.md) 문서를 참고합니다.

---

## 6. 소프트웨어 연동 기준

본 문서에 정의된 하드웨어 제원은 다음 모듈에서 참조됩니다.

- **Spine**: 기구학 계산, 관절 각도 검증
- **Sketch**: 로봇 위치 및 드로잉 시각화
- **Dashboard**: 사용자 입력 제한 및 실시간 제어

향후 실제 하드웨어 제어 모듈이 추가될 경우,  
본 문서의 기준을 그대로 사용하여 제어 일관성을 유지합니다.

---

본 문서는 하드웨어 확장에 따라 지속적으로 갱신될 수 있습니다.
