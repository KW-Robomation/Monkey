# Terminology

## 1. 개요

본 문서는 로봇팔 시뮬레이터 프로젝트 **Plutto**에서 사용되는  
주요 용어 및 개념을 정리한 문서입니다.

본 문서에 정의된 용어들은  
Plutto 시스템의 소프트웨어 구조, 로봇 동작 생성 과정,  
통신 포맷을 설명하는 전 문서 전반에서 **별도의 추가 설명 없이 사용**됩니다.

---

## 2. SVG 및 경로

### 2.1. SVG (Scalable Vector Graphics)

XML 기반의 벡터 이미지 포맷으로,  
점, 선, 곡선 등의 기하학적 정보로 이미지를 표현합니다.

Plutto 시스템에서는 SVG 파일을 입력으로 받아  
로봇 드로잉 경로를 생성합니다.

---

### 2.2. 경로 (Path)

SVG 내에서 실제 형상을 정의하는 요소로,  
`<path>` 태그의 `d` 속성에 문자열 형태로 저장됩니다.

경로는 이동(M), 선(L), 수평·수직 이동(H, V), 닫기(Z)  
등의 명령어로 구성됩니다.

---

## 3. 로봇 구조 및 기구학

### 3.1. 기구학 (Kinematics)

로봇의 관절 각도와 링크 구조를 기반으로,  
로봇 말단(End Effector)의 위치와 자세를 계산하는 이론입니다.

Plutto 시스템에서는 로봇 동작 생성의 핵심 계산 과정으로 사용됩니다.

---

### 3.2. 정기구학 (Forward Kinematics)

관절 각도가 주어졌을 때,  
로봇 말단의 위치를 계산하는 과정입니다.

시뮬레이션 및 시각화 단계에서  
로봇의 현재 상태를 표현하기 위해 사용됩니다.

---

### 3.3. 역기구학 (Inverse Kinematics)

로봇 말단의 목표 위치가 주어졌을 때,  
이를 만족하는 관절 각도를 계산하는 과정입니다.

SVG 경로의 좌표를  
로봇 관절 각도 기반의 동작 데이터로 변환하는  
핵심 계산 단계입니다.

---

## 4. 좌표 및 동작

### 4.1. 좌표계 (Coordinate System)

로봇 동작 계산 및 시각화를 위해 정의된 기준 좌표 체계입니다.

Plutto 시스템 내부에서는  
SVG 좌표계와 로봇 좌표계 간 변환을 수행하며,  
모든 계산은 일관된 좌표계를 기준으로 수행됩니다.

---

### 4.2. 펜 업 / 다운 (Pen Up / Pen Down)

로봇 말단에 장착된 펜의 상태를 의미합니다.

- **Pen Up**: 펜이 종이에서 떨어진 상태로, 이동만 수행
- **Pen Down**: 펜이 종이에 접촉한 상태로, 드로잉 수행

펜 상태는 관절 동작 정보와 함께  
동작 데이터에 명시적으로 포함됩니다.

---

### 4.3. 관절 1 (Joint 1)

로봇팔의 첫 번째 회전 관절로,  
로봇 전체의 좌우 회전 및 스케치 범위를 결정합니다.

기구적으로는 어깨(Shoulder)에 해당하는 역할을 합니다.

---

### 4.4. 관절 2 (Joint 2)

로봇팔의 두 번째 회전 관절로,  
말단부의 전후 이동 및 스케치 범위를 결정합니다.

기구적으로는 팔꿈치(Elbow)에 해당하는 역할을 합니다.

---

## 5. 데이터 및 통신

### 5.1. 동작 데이터 (Motion Data)

로봇의 관절 각도 변화와 펜 상태로 구성된  
연속적인 실행 단위의 데이터입니다.

Plutto 시스템에서는 JSON 기반 구조 또는  
Plutto Path 포맷으로 관리됩니다.

---

### 5.2. Plutto Path

Plutto 프로젝트에서 정의한  
**1바이트 기반 로봇 동작 통신 포맷**입니다.

관절 스텝 변화량과 펜 상태 제어를  
고정 크기 데이터로 압축하여 표현합니다.

자세한 구조는  
[Plutto Path 문서](Plutto_Path.md)를 참고합니다.
